## 1. 프로젝트 개요

### 프로젝트명
**얼마니 (Eolmani)**

### 한 줄 정의
공공 데이터(KAMIS, 참가격)와 기상청 예보를 결합해 **식료품 가격을 추적하고, 언제 사야 이득인지 알려주는** 무료 웹 서비스. 김장/장아찌/제철 등 한국 식문화의 대량 구매 타이밍에 특화.

### 슬로건
- **"오늘 사과 얼마니?"** — 가격을 묻는 본질적 질문
- **"미리 사두면 절반 가격"** — 타이밍 가치 강조
- **"날씨가 알려주는 다음 주 가격"** — 예측 기반 차별점

### 이름의 의미
"얼마니?" — 가격을 묻는 가장 본질적인 질문. 서비스의 모든 기능이 이 질문 하나로 통합됨.

- **오늘 사과 얼마니?** → 가격 표시
- **다음 주에는 얼마니?** → 날씨 기반 예측
- **김장철엔 얼마니?** → 시즌 대량구매 캘린더
- **g당 얼마니?** → 슈링크플레이션 (보류)
- **내 물가 얼마니?** → 개인화 CPI (보류)

### 비전
정부와 공공기관이 보유한 가격 데이터는 충분하지만, 일반 소비자가 쉽게 활용할 수 있는 도구는 부족하다. 얼마니는 정보 비대칭을 해소하고, 누구나 자신의 소비를 객관적으로 평가하고 **합리적 구매 타이밍**을 잡을 수 있는 무료 도구를 제공한다.

## 2. 문제 정의

### 친구 피드백에서 발견한 진짜 문제 (2026-04-28)

> "지금 데이터는 약간 소비자 물가지수 계산용 느낌. 실용성이 좀 없는 느낌."
> "마트마다 가격 업데이트가 다르고 지역마다 차이도 있을 텐데."
> "일기예보 기반으로 가격 예측해서 어떤 거 미리 사놔야 하는지 알려주면 좋겠다.
> 김치나 장아찌 같은 대용량 반찬 만드는 집이 많아서 가격 쌀 때 많이 사놔야 함."

### 진짜 풀어야 할 문제

1. **"언제 사야 이득인지 모름"** — 평년 대비 비싼지 싼지 판단 어려움
2. **"미리 못 사서 비싸게 삼"** — 김장철, 제철 시기 놓침
3. **"날씨가 가격에 미치는 영향을 모름"** — 폭염/한파/장마 후 가격 폭등 못 피함
4. **"대량 구매 타이밍 정보 부재"** — 김치/장아찌 만드는 가정의 핵심 니즈
5. **공공 사이트 UX 문제** — KAMIS·참가격은 데이터는 좋지만 UI가 투박, 모바일 경험 나쁨

### 얼마니가 해결하는 것

KAMIS의 평균가 데이터는 마트별 비교에는 부족하지만 **트렌드 분석과 타이밍 가이드에는 정확히 적합**. 거기에 기상청 예보 + 시즌 인사이트를 결합해 **"지금이 살 때인지" 신호**를 제공.

데이터 출처와 한계는 명확히 라벨링하여 신뢰성 확보. 향후 영수증 OCR로 사용자 실측 데이터까지 누적되면 마트별 비교 영역도 보강.

### 5-1. 기본 가격 표시 + 변동률 ✅ (Week 1 완료)

주요 식재료 20종의 가격 + 7일/30일 변동률 + 카테고리별 분류 + 오늘의 특가 TOP5. KAMIS 데이터 매일 06:00 자동 수집.

### 5-2. 30일 라인 차트 (Chart.js) — 이번 주

품목 카드 클릭 시 상세 페이지로 이동. 30일치 가격 추이를 라인 차트로 표시. 메인 카드에는 미니 스파크라인으로 한눈에 추세 파악.

### 5-3. 메인 카피 수정 — 이번 주

데이터 정체성을 명확히 라벨링하여 친구가 지적한 "마트 가격으로 오해되는" 문제 해결.

❌ "오늘 사과 가격 25,000원" (오해 유발)
✅ "전국 평균 25,000원 | KAMIS 도소매 조사 평균 | 트렌드 지표"

### 5-4. "지금이 살 때?" 신호등 + 통합 카드 — Week 2

KAMIS 응답의 dpr1~dpr7로 즉시 구현 가능. 4가지 요소 한 카드에 통합:
- **숫자** (정량 신뢰): 평년/1주일/1개월 변화 % + 원
- **신호등** (직관 판단): 🟢 살 때 / 🟡 평균 / 🔴 비쌈
- **절약금액** (행동 임팩트): "1박스 사면 6,000원 절약 기대"
- **행동유도** (다음 행동): "다음 주 한파 예보 → 미리 구매"

**판단 로직 (가중치 방식):**
- 평년 대비 50% + 1개월 변화 30% + 1주일 변화 20%
- 종합 점수 ≤ -5% → 🟢 / -5%~+5% → 🟡 / ≥ +5% → 🔴

### 5-5. 시즌 대량구매 캘린더 — Week 2

월별 자동 인사이트로 "지금 무엇을 미리 사야 하는지" 안내.

- **1~2월**: 한라봉, 딸기 시즌
- **3~4월**: 쑥, 봄나물
- **5월**: 매실 (장아찌)
- **6~7월**: 마늘, 양파 (1년치 비축)
- **8월**: 복숭아 시즌
- **9~10월**: 추석 차례상 + 사과/배 출하
- **11월**: 김장 시즌 ⭐
- **12월**: 동태/명태 (제사상)

### 5-6. 일일 브리핑 카드 — Week 2~3

매일 첫 화면 상단에 신문 헤드라인 같은 한 줄 + 핵심 시그널 3개. 룰 기반으로 데이터 추출 후 GPT-4o-mini로 자연스러운 문장 생성. 매일 표현이 다양해서 재방문 동기 형성.

```
🗞️ 오늘의 얼마니 (4/28 월요일)
☀️ 서울 22℃ 맑음

이번 주 후반 한파 예보 → 잎채소 5종 가격 상승 우려
오늘 가성비 TOP 3:
🟢 사과 25,000원 (-12% 평년 대비)
🟢 무 1,200원 (-8%)
🟡 양파 3,400원 (보합)
```

### 5-7. 기상청 중기예보 API + 가격 영향 — Week 3

**기상청 중기예보** (10일치, 무료 공공데이터)로 7일치 일~토 날씨 카드를 가로 스크롤 형태로 표시. 각 날짜에 영향 받을 품목 표시.

```
[일] [월] [화] [수] [목] [금] [토]
☀️   ☀️   🌧️   🌧️   ❄️   ❄️   ☀️
22°  20°  18°  15°   5°   3°   8°
              🥬↑   🥬↑↑  🥬↑
              🍎↓   🍎↓
```

**날씨 → 가격 영향 룰 테이블** (사전 정의, 농업기술센터 자료 기반):
- ❄️ 한파 (-5℃ 이하): 잎채소 +15% (배추, 시금치, 상추)
- ☀️ 폭염 (33℃ 이상 3일+): 잎채소 +20%, 여름과일 -10%
- 🌧️ 폭우 (80mm 이상): 노지 채소 +10%
- 🌫️ 장마 (5일+): 오이/호박/토마토 +12%

### 5-9. 지역별 가격 지원 (Phase 22) — 다음 목표

현재 모든 가격은 **서울(1101) 소매가** 기준 — 지방 사용자는 체감과 괴리가 있음. KAMIS API의 `p_country_code` 파라미터를 활용해 사용자가 자기 지역 가격을 볼 수 있게 한다.

**왜 중요한가:**
- 지역별 가격 차이가 실제로 큼 (제주·강원 등 일부 품목 10~30% 차이)
- "내 지역 기준"이라는 개인화는 재방문 동기를 만드는 핵심 요소
- KAMIS가 이미 데이터를 제공 — 구현 난이도 대비 임팩트 높음

**KAMIS p_country_code 전체 지원 지역:**

소매가격 지원 24개:

| 코드 | 지역 | 코드 | 지역 | 코드 | 지역 |
|------|------|------|------|------|------|
| `1101` | 서울 | `2100` | 부산 | `2200` | 대구 |
| `2300` | 인천 | `2401` | 광주 | `2501` | 대전 |
| `2601` | 울산 | `2701` | 세종 | `3111` | 수원 |
| `3112` | 성남 | `3113` | 의정부 | `3138` | 고양 |
| `3145` | 용인 | `3211` | 춘천 | `3214` | 강릉 |
| `3311` | 청주 | `3411` | 천안 | `3511` | 전주 |
| `3613` | 순천 | `3711` | 포항 | `3714` | 안동 |
| `3814` | 창원 | `3818` | 김해 | `3911` | 제주 |

도매가격 지원 5개: 서울·부산·대구·광주·대전

> `''`(빈 문자열) = 전체(전국 평균), default. 현재 구현: 전국 평균 + 6대 도시.

**현재 구현된 지역 (1차, 6대 도시 + 전국):**

| 코드 | 지역 |
|------|------|
| `''` | 전국 평균 (기본값) |
| `1101` | 서울 |
| `2100` | 부산 |
| `2200` | 대구 |
| `2300` | 인천 |
| `2401` | 광주 |
| `2501` | 대전 |

**구현 방향:**
1. `price_history`에 `region_code VARCHAR(10)` 컬럼 추가 (Alembic 마이그레이션)
2. `price_collector.py` — 6개 지역 순회 수집 (APScheduler 06:00 KST, 기존 수집 후 추가)
3. `price_service.py` — `region_code` 파라미터 추가, 기본값 `'1101'`
4. 헤더에 지역 선택 드롭다운 (localStorage 유지)
5. 상세 페이지 — "서울보다 X% 비쌈/저렴" 비교 뱃지 (선택 사항)

**주의:**
- API 호출 6배 증가 → KAMIS 요청 제한 확인 필요
- PriceHistory Unique constraint: `(item_id, recorded_date, source)` → `(item_id, recorded_date, source, region_code)` 조정
- 지역 데이터 없는 날 → 서울 폴백 or `None` 처리

### 5-8-b. 공산품 별도 섹션 — Week 4~5

농수산물(KAMIS, 타이밍 위주)과 별도 영역으로 공산품 추가. 데이터 소스: **참가격 (price.go.kr, 한국소비자원)**.

공산품 카드는 신호등 대신 **마트별 비교** 위주:
```
🥤 코카콜라 1.5L
최저가 3,150원 (홈플러스) / 평균 3,450원 / 최고가 3,890원
이마트 3,280원 ↘ / 홈플러스 3,150원 → ⭐ / 롯데마트 3,500원 ↗
```

### 5-10. 가격 알림 — 후순위 보류

회원가입 + 관심 품목 즐겨찾기 + 목표 가격 설정. 알림 채널 단계적 도입:
1. **이메일** (1차, 무료, 즉시 가능)
2. **카카오 "나에게 보내기"** (2차, 무료, 사업자등록 불필요, 한국 사용자 친화)
3. **카카오톡 채널 친구톡** (3차, 사용자 1,000명+ 시 검토, 비용 발생)

### 5-11. 보류 중인 기능들

다음 기능들은 정체성·우선순위 재검토 결과 보류:

- **영수증 OCR**: 사용자 1,000명+ 모일 때 재검토. 그때부터 실제 마트 가격 데이터 의미 있어짐.
- **개인화 CPI**: 영수증 데이터 의존. OCR 도입 후 재검토.
- **슈링크플레이션 감지**: 가공식품 이슈인데 현재 KAMIS는 농수산물 위주. 공산품 섹션(Week 4~5) 안정화 후 재검토.
- **AI 가격 예측**: 룰 기반 예측이 부족해질 때 도입. 1년치 데이터 누적 후.
- **쿠팡 파트너스**: 사용자 1,000명+ 시 도입. 평균가 대비 합리적인 가격만 추천하는 정책으로 신뢰 유지.

## 9. 비즈니스 모델

### 운영 원칙
- **무료 서비스**: 광고 없음, 유료 결제 없음
- **비상업 운영**: 공익성 강조, "정보 비대칭 해소" 사회적 가치
- **후원 기반**: 토스 송금하기 / Buy Me a Coffee 링크 (Footer)
- **간접 수익**: 포트폴리오 가치, 취업 레퍼런스, 컨설팅 기회

### 왜 이 모델인가
식료품 가격 추적은 본질적으로 광고·제휴로 큰 수익을 내기 어렵다. 사용자 신뢰가 핵심이고, 광고를 붙이면 객관성이 훼손된다. 대신 "믿을 수 있는 무료 서비스"로 포지셔닝하면 사용자가 모이고, 그 자체가 만든 사람의 자산이 된다. 수익은 별도 자동화 프로젝트에서 만든다.

향후 사용자 1,000명+ 시 쿠팡 파트너스 도입은 검토하되, **평균가 대비 합리적인 가격만 추천**하는 정책으로 신뢰 유지.

## 16. 친구 피드백 심층 분석 (2026-05-01)

> 서비스 오픈(www.eolmani.com) 직후 받은 두 번째 친구 피드백.
> 1차(2026-04-28)가 "데이터 정체성"을 흔들었다면, 2차는 **"누가, 왜, 어떻게 쓸 것인가"** — 타겟·UX·행동 전환을 흔든다.

### 16-1. 피드백 핵심 요약 — 표면 불만 vs 근본 문제

| 표면 불만 | 근본 메시지 |
|----------|-----------|
| "단위가 안 적혀있다" | **정보의 actionability 결여** — "이 숫자를 어떻게 써먹으라는 거냐" |
| "마트랑 다른데 의미 없음" | **정보→행동 경로의 끊김** — "보고 갔더니 틀리면 신뢰가 깨진다" |
| "쿠팡 연결되면 좋겠다" | **One-stop 욕구** — 정보와 구매를 한 화면에서 끝내고 싶음 |
| "아줌마들이 보겠냐" | **페르소나 검증 요구** — 타겟이 진짜 생각한 사람들이 맞냐 |
| "주식 같다, 전단지처럼" | **톤앤매너 미스매치** — 데이터 보는 사람보다 가볍게 훑는 사람이 다수 |
| "자취생은 배민" | **시장 사이즈 의심** — 표본 1명의 라이프스타일을 시장 전체로 일반화 |

**친구 피드백의 진짜 가치**: "타이밍을 알려주고 끝나는 게 아니라 그 자리에서 살 수 있게 해줘야 한다" — 정보 제공형(informational)에서 행동 유발형(transactional)으로 나아가라는 요구.

### 16-2. 수용할 피드백 (즉시 반영)

#### 16-2-1. 단위 표기 명확화 [P0, ~1일]
**문제**: "김 10장이 어떤 크기 10장인지 모름", "조기 마리당? 100g당?"
**조치**:
- 가격 옆에 단위 강조 표기: `10,800원 / 1kg`, `4,500원 / 10장`
- 김·생선처럼 단위가 모호한 품목에 보충 설명 추가 (`조기: 1마리(250~300g 기준)`)
- `Item` 테이블에 `unit_description` 컬럼 추가 또는 `unit` 표기 강화

#### 16-2-2. 정렬 기본값 = "🟢 신호 우선" [P0, ~0.5일]
**문제**: 카드가 등록순으로 나와 "뒤죽박죽" 느낌
**조치**:
- 기본 정렬: **신호등(🟢→🟡→🔴) → 평년 대비 변동률 오름차순**
- 정렬 드롭다운 추가: 추천(신호등) | 변동률 | 이름순
- "싼 것부터" 직관 ≠ "지금 살 때부터" — 차이를 UI에서 명확히

#### 16-2-3. 가격 기준 배지 추가 [P0, ~0.5일]
**문제**: "도매? 소매? 시장? 마트?" — 푸터에만 있고 카드에서 안 보임
**조치**:
- 모든 가격 옆 소형 배지: `소매 평균` 추가
- 상세 페이지 상단: "이 가격은 [지역] 소매가 평균입니다. 마트별 ±10~20% 차이 있을 수 있습니다." 명시

#### 16-2-4. 글씨 크기 상향 [P0, ~0.5일]
**문제**: "아줌마 할머니가 못 봄"
**조치**:
- 본문 폰트: `text-sm(14px)` → `text-base~lg(16~18px)`
- 가격 숫자: `text-xl~2xl(20~24px)` + 굵게
- 라벨/캡션: 14px 유지, 색 대비 강화 (gray-500 → gray-700)

### 16-3. 부분 수용할 피드백 (방향은 맞지만 방식 조정)

#### 16-3-1. "쿠팡/네이버 연결" → 검색 링크로 1차 즉시 도입
**친구 요구**: 해당 가격에 맞는 쿠팡 상품 직접 연결
**조정안**:
- **Phase A (즉시, 수익 없음)**: 카드 우측 하단 외부 검색 링크 버튼
  - `[쿠팡에서 보기]` → `coupang.com/np/search?q={품목명}`
  - `[네이버쇼핑]` → `search.shopping.naver.com/search/all?query={품목명}`
  - 푸터 고지: "외부 링크는 참고용이며 제휴 관계 없음"
- **Phase B (사용자 1,000명+)**: 쿠팡 파트너스 ID 추가 (기존 URL에 `lptag` 추가만 하면 됨), 수수료 고지문 추가
- **Phase C (사용자 5,000명+)**: 파트너스 상품 검색 API로 카드에 상품 1~3개 임베드

#### 16-3-2. "마트에서 다르게 팔면 의미 없다" → 데이터 정체성 재라벨링
**현실**: KAMIS는 도소매 조사 평균 — 개별 마트 가격이 아님
**조정안**:
- 신선식품: "가격 추세/타이밍 지표"로 포지셔닝 강화 (정확한 가격 아닌 비싼지/싼지 판단 도구)
- 공산품(Week 4~5): 참가격 기반 마트별 실제 가격 비교 가능 → 이 영역에서 친구 요구 충족
- 메인 페이지에 두 영역 시각적 구분: "📊 가격 트렌드 (농수산물)" / "🏪 마트별 비교 (공산품)"

#### 16-3-3. "전단지처럼" → 히어로 카드 부분 도입
**조정안**:
- 기본 디자인 신뢰감 유지 (데이터 서비스 본질 훼손 금지)
- 메인 최상단에 🟢 신호 품목 1~3개를 큰 가격 + "X원 절약" 강조하는 전단지형 히어로 카드 추가
- 노란 배지·빨간 가격 강조 등 마트 전단지 악센트 부분 도입

### 16-7. 참가격 API 검토 (Week 4~5 공산품 섹션용)

> 사용자가 공유한 링크(data.go.kr/data/15098486)는 KAMIS **가공식품** 데이터.
> **참가격 공식 API는 별도 데이터셋 (ID: 3043385)**

| 항목 | 내용 |
|------|------|
| API 엔드포인트 | `openapi.price.go.kr/openApiImpl/ProductPriceInfoService/` |
| 라이선스 | 공공누리 1유형 — **상업적 이용 가능, 출처 표시 필수** |
| 비용 | 무료 (자동 승인) |
| 조사 품목 | 생필품 120~168개 (신선채소 전 품목 아님) |
| 업데이트 주기 | 격주 (2주) — 실시간성 낮음 |
| 데이터 포맷 | XML |
